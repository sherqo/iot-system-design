\section{Application Development}

\subsection{Application Overview and Platform}
The application layer of the IoT system consists of a web-based dashboard that serves as the main interface between users and the system. It provides a centralized platform for interacting with real-time data collected from sensors and controlling connected devices.

Its main purpose is to enable effective monitoring and management of the IoT environment by presenting live information in a clear and organized way. The dashboard acts as the primary operational tool for users, allowing them to observe system status, respond to alerts, and manage device behavior efficiently.

The application layer is a web-based dashboard acting as the primary interface between users and the IoT system. It centralizes real-time data visualization and device control in one operational view.

\subsection{User Role}
The system is designed exclusively for admin users, who are responsible for monitoring and managing the IoT environment through the web-based dashboard. The admin has full access to real-time sensor data, system status, and device controls. Their role includes overseeing environmental conditions, responding to alerts, and adjusting system settings when necessary to ensure optimal performance and safety.

The system is designed for an admin role with full access to monitoring, alert handling, and override control to maintain system performance and safety.

\subsection{Core Features}
The web-based dashboard provides a set of essential functionalities that enable effective monitoring and control of the IoT system, including the following core features:
\begin{itemize}
  \item Real-time monitoring of sensor data (e.g., CO2, temperature, humidity, occupancy)
  \item Live system status display for all connected devices
  \item Alert and notification system for abnormal conditions
  \item Control of IoT actuators (e.g., ventilation system)
  \item Overview of historical data visualization and basic trends (e.g., peak hours)
\end{itemize}

The web-based dashboard communicates with the IoT system through a backend layer that enables real-time data exchange and control. It continuously receives updated sensor readings and system statuses, while also sending user commands to connected devices for execution.

\textbf{Application-system interaction:}
\begin{itemize}
  \item Retrieves real-time data from IoT sensors via backend APIs
  \item Displays continuously updated system status on the dashboard
  \item Sends control commands to actuators (e.g., ventilation system)
  \item Supports bidirectional communication for monitoring and control
  \item Ensures low-latency updates for real-time system responsiveness
\end{itemize}
